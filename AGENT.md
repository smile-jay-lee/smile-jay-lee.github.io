# AGENT 提示文档（优化版）

当你在本仓库中提交修改时，请遵循以下约定以保持提交记录清晰且可追溯：

1. 提交流程（命令行形式）
   - 在仓库根目录执行：

     ```powershell
     git add .
     git commit -m "<type>(<scope>): <简短描述>  说明修改内容与理由"
     git push
     ```

2. 提交信息要求
   - 提交信息要反映具体功能/修改点，例如：
     - `feat(docs): 优化 AGENT.MD 提示，增加提交示例`
     - `fix(js): 修复 dynamic-mode.js 中的 toast 样式`
   - 若修改了特定文件，请在消息中注明（例如：`修改文件：AGENT.MD`）。

3. 额外建议
   - 小而单一的改动请使用独立提交，便于回溯与回滚。
   - 在变更较大或涉及多组件时，建议先创建分支并在 PR 描述中写明变更范围与验证步骤。

如果你希望，我可以代替你执行提交（已读文件并生成合适的提交信息），或将提交示例自动化成脚本。
